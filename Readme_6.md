# Лабораторная работа №6 – Проектирование Event-Driven архитектуры

В рамках данной работы для системы управления складом (вариант №18) спроектирована событийно-ориентированная архитектура с применением брокера сообщений RabbitMQ и паттерна CQRS.

## Что сделано

- Проанализированы бизнес-операции и выделены ключевые **события** (`UserCreated`, `ProductCreated`, `ProductUpdated`, `ReceiptCreated`, `InventoryWriteOff`, `InventoryStockChanged`).
- Определены **производители** (основной API-сервис `inventory-api`) и **потребители** событий (сервисы уведомлений, аналитики, обновления read-модели).
- Спроектирован **поток событий** через RabbitMQ: topic exchange, маршрутизация по ключам `event.<EventType>`, гарантии доставки **at-least-once**.
- Применён паттерн **CQRS**: команды (write side) оставлены в PostgreSQL, а для чтения предложена денормализованная read-модель, обновляемая асинхронно через события.
- Сформирован **каталог событий** с описанием payload, маршрутизации и потребителей.

## Документация

- **`event_driven_design.md`** – полное описание Event-Driven архитектуры: анализ событий, компоненты, формат сообщений, гарантии доставки, применение CQRS, диаграмма потоков.
- **`event_catalog.md`** – сводная таблица всех событий с указанием производителей, потребителей, структуры payload и гарантий.
