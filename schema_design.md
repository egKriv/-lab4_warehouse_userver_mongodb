
# Schema Design (MongoDB)

## Выбор коллекции
Для хранения истории списаний выбрана MongoDB, так как:
- События списания — это журнал операций, не требующий строгих транзакционных гарантий.
- MongoDB хорошо подходит для хранения событий и быстрой агрегации.
- Гибкая схема позволяет легко добавлять новые поля (причина, автор и т.д.).

## Структура документа
Коллекция: `writeoffs`
```json
{
  "_id": ObjectId,
  "product_id": "UUID строкой",
  "quantity": int,
  "reason": string,
  "created_at": "ISO8601 дата-время"
}
```

## Связь с PostgreSQL
`product_id` хранится как строка (UUID), ссылаясь на таблицу `products` в PostgreSQL. Выбран reference, а не embedded, чтобы избежать дублирования данных о товаре и сохранить консистентность с основной БД.
