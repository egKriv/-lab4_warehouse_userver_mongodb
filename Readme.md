# Inventory Management API (Вариант 18)

REST API для управления складом на базе Yandex Userver с хранением данных в PostgreSQL.  
Проект включает схему БД, индексы, примеры запросов и анализ оптимизации.

## Структура проекта

- `configs/` – конфигурационные файлы сервиса
- `postgres/init.sql` – скрипт инициализации БД (выполняется при первом запуске контейнера)
- `schema.sql` – создание таблиц и индексов
- `data.sql` – тестовые данные (минимум 10 записей на таблицу)
- `queries.sql` – SQL-запросы для всех операций предметной области
- `optimization.md` – описание индексов и сравнение планов `EXPLAIN`
- `src/` – исходный код C++ (контроллеры, сервисы, хранилище, JWT-аутентификация)
- `tests/` – тесты API и скрипты для ручного тестирования
- `Dockerfile`, `docker-compose.yml` – контейнеризация приложения вместе с PostgreSQL

## Схема базы данных

База данных состоит из трёх таблиц:

- **users** – пользователи системы (логин, пароль, роль)
- **products** – товары на складе (название, артикул, количество, единица измерения)
- **receipts** – поступления товаров (связь с продуктом, количество, поставщик, автор)

Все первичные ключи – UUID. Созданы индексы для ускорения поиска по логину, email, названию товара, фильтрации поступлений по продукту и дате. Подробное обоснование приведено в [`optimization.md`](optimization.md).
